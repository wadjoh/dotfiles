# Chezmoi Dotfiles

Cross-platform dotfiles managed with [chezmoi](https://www.chezmoi.io/), targeting **macOS** and **Arch Linux**.

## Chezmoi Naming Conventions

- `dot_` prefix → `.` (e.g., `dot_zshrc` → `.zshrc`)
- `private_` prefix → file/dir with `0o700`/`0o600` permissions
- `create_` prefix → only created if the target doesn't already exist
- `symlink_` prefix → creates a symlink
- `.tmpl` suffix → processed as a Go template
- `run_` prefix → executable scripts, not managed files
  - `run_once_` → runs once (tracked by content hash)
  - `run_onchange_` → runs when script content changes
  - `run_after_` → runs after file updates
  - `run_once_before_` → runs once, before file updates
  - Alphabetical ordering controls execution order (e.g., `zzz-` prefix runs last)

## Repository Structure

```
.chezmoi.yaml.tmpl        # Chezmoi config (merge/edit/diff tool settings)
.chezmoiignore            # Platform-conditional file exclusions
dot_zshrc                 # Main shell config (platform-aware)
dot_p10k.zsh              # Powerlevel10k config (AUTO-GENERATED — do not edit)
dot_gitconfig             # Git config
dot_Brewfile              # Homebrew packages (macOS + shared)
dot_vscode.Brewfile       # VS Code extensions
create_dot_custom.zshrc   # Per-machine zsh overrides (created once, never overwritten)
dot_config/               # XDG configs (bat, btop, ghostty, micro, atuin, paru)
private_Library/          # macOS-only app configs (VS Code, iTerm2 symlinks)
run_*                     # Setup scripts (see below)
```

## Platform Gating

`.chezmoiignore` uses Go template conditionals with these variables:
- `.chezmoi.os` — `"darwin"` or `"linux"`
- `.chezmoi.osRelease.id` — `"arch"` on Arch Linux

Current rules:
- `private_Library/` → macOS only
- `dot_config/paru/` → Arch Linux only

When adding platform-specific files, always add a corresponding rule in `.chezmoiignore`.

## Run Scripts

| Script | When | Platform | Purpose |
|--------|------|----------|---------|
| `run_once_before_install-brew.sh` | Once, before apply | Any | Install Homebrew |
| `run_after_brew-bundle.sh` | After apply | Any (skips if no brew) | `brew bundle --global` + VS Code extensions |
| `run_after_arch-setup.sh` | After apply | Arch Linux | zsh default shell, Docker, Ghostty, keyd |
| `run_onchange_macos-defaults.sh` | On change | macOS | MiddleClick config |
| `run_after_zzz-set-configs.sh` | After apply (last) | macOS | macOS system defaults |

## Verification Commands

```sh
chezmoi diff          # Preview changes before applying
chezmoi apply -n      # Dry-run apply
chezmoi apply         # Apply changes
chezmoi managed       # List all managed files
```

## Constraints

- **Do not edit `dot_p10k.zsh`** — it is auto-generated by `p10k configure`
- **Do not edit `create_dot_custom.zshrc`** content after creation — it's a per-machine override file that chezmoi only creates once
- Always gate platform-specific files in `.chezmoiignore`
- This repo uses Rose Pine theming across tools (btop, micro, ghostty) — maintain consistency when adding new tool configs
